# 部署说明

## 部署方式

推荐使用 Docker 进行部署，内置 PostgreSQL 数据库，无需外部依赖。

```bash
# 复制并编辑环境变量
cp .env.example .env

# 构建并启动所有服务
docker compose up -d
```

启动后访问 http://localhost:3000 即可使用。

详见 [README](https://github.com/jihe520/mindpocket#-deploy) 获取完整的 Docker 部署指南。

## 站点拆分

这个仓库当前包含两个主要 Web 入口：

- `apps/site` 提供官网与 `/docs`
- `apps/web` 提供登录后的产品能力

可以分别部署，也可以按你的基础设施策略合并处理。

## 部署前检查

1. 确认 `pnpm install` 成功
2. 配置数据库连接与认证环境变量
3. 为 `apps/web` 完成数据库初始化
4. 验证 `pnpm build` 或按应用分别构建

## 数据库相关

开发阶段可使用：

```bash
pnpm --filter web db:push
```

生产环境建议走迁移流程，而不是直接推送 schema。
